# 产品概览

## 定位

DeVault 提供 **统一控制面 + 边缘 Agent + 对象存储数据面** 的备份与恢复能力，适用于希望 **自托管或使用专属部署**，同时需要 **多租户隔离、API 优先集成与可观测性** 的组织。产品线以 **可订阅、可规模化交付的 SaaS 形态**为演进目标：同一套内核支持云多租户服务与专用环境。

## 核心价值

- **控制面与数据面分离**：作业元数据、租约与预签名签发在控制面；备份对象经 HTTPS 写入租户隔离的存储前缀，**大对象不经 gRPC 传输**。
- **Pull 模型**：Agent 主动连接平台 gRPC，便于客户侧 **仅开放出站**、穿越 NAT 与防火墙。
- **多租户与安全基线**：租户作用域 API、RBAC、可选 OIDC、按租户的存储与合规选项（详见 [租户与访问控制](../admin/tenants-and-rbac.md)）。
- **可运维与可售卖**：Prometheus 指标、健康检查、版本与能力协商、gRPC 审计日志挂钩点；便于包装 **SLA、配额与计费** 信号（见访问控制文档中的用量指标）。

## 典型场景

- 文件与目录级 **全量备份与时间点恢复**，配合 Cron 调度与保留策略。
- 与 **MinIO / AWS S3 / 兼容实现** 集成，支持 **STS AssumeRole** 与 **客户专用桶（BYOB）** 等部署模式。
- **恢复演练（restore drill）** 周期性校验备份可读性与完整性。
- 在 **网关后水平扩展** 控制面 gRPC，满足企业入口与审计要求。

## 能力边界

当前主线为 **文件类插件**、**作业与 Artifact 元数据模型**、**可审计的 S3 数据面**；不提供「全品类工作负载一站式备份套件」的等同能力。数据库增量备份等可作为后续插件扩展，须保持控制面与 Agent 职责边界（见 [路线图](./roadmap.md)）。

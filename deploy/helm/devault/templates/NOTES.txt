1. Build the application image from the repository root (context must include pyproject.toml, src/, alembic/):

   docker build -f deploy/Dockerfile -t {{ .Values.image.repository }}:{{ .Values.image.tag }} .

2. If you use a private registry, retag/push and set image.repository / image.tag accordingly.

3. Wait for the API pod to become Ready, then port-forward HTTP:

   kubectl wait --for=condition=available deployment/{{ include "devault.fullname" . }}-api --timeout=300s
   kubectl port-forward svc/{{ include "devault.fullname" . }}-api 8000:8000

4. Call the health endpoint:

   curl -sf http://127.0.0.1:8000/healthz

5. HTTP access uses IAM-issued JWTs: configure DEVAULT_IAM_JWT_* on the API deployment (not set in this demo chart — API runs in dev-open mode until you add them).

6. Agent gRPC (in-cluster): {{ include "devault.fullname" . }}-api:50051 — set DEVAULT_GRPC_REGISTRATION_SECRET to match the same secret key on agents outside the chart.

7. bundled PostgreSQL / Redis / MinIO are for lab use only; run a single scheduler replica (already enforced).

   `docker build -f deploy/Dockerfile.console -t {{ .Values.console.image.repository }}:{{ .Values.console.image.tag }} .`
   then `kubectl rollout status deploy/{{ include "devault.fullname" . }}-console`.
   Ingress on the chart host splits `/api`, `/docs`, … to API and `/` to the console (same origin as nginx in the image).
